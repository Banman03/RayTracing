Motivation: What Must a Renderer Simulate?

  • Direct illumination - how a surface responds to a light source
  • Shadows - hard (point light) or soft (area light)
  • Specular reflections - highlights, mirror inter-reflections
  • Indirect illumination - light bouncing off diffuse surfaces
  • Caustics - focused light through/off curved surfaces

The Rendering Equation  Kajiya, 1986

Reflected radiance at surface point \(\vec{x}\) in direction \(\vec{\omega}\):

\[ L_o(\vec{x}, \vec{\omega}) = L_e(\vec{x}, \vec{\omega}) + \int_\Omega f_r(\vec{x},\,\vec{\omega}_i \to \vec{\omega})\; L_i(\vec{x}, \vec{\omega}_i)\;\cos\theta_i\; d\omega_i \]
  • \(L_e\) - emitted radiance (light sources)
  • \(f_r\) - BRDF: probability that incoming \(\omega_i\) scatters into \(\omega\)
  • \(L_i(\vec{x},\vec{\omega}_i) = L_o(\vec{x}', -\vec{\omega}_i)\) - radiance arriving from direction \(\omega_i\)
  • \(\cos\theta_i\) - Lambert foreshortening

\(L_i(\vec{x},\vec{\omega}_i)\) is the outgoing radiance \(L_o\) of whatever surface is visible from \(\vec{x}\) in direction \(\vec{\omega}_i\) - so \(L\) appears on both sides of the equation. This self-referential structure makes it a Fredholm integral equation of the second kind: the unknown function is part of the integrand, and it cannot be solved by direct evaluation.

Every algorithm in this talk is an approximation or estimator of this equation.

Ray Fundamentals

Ray parameterization:

\[ \vec{P}(t) = \vec{o} + t\,\vec{d}, \quad \|\vec{d}\| = 1 \]

Ray–sphere intersection:

\[ a t^2 + b t + c = 0 \]

\(a = \vec{d}{\cdot}\vec{d},\; b = 2(\vec{o}-\vec{c}){\cdot}\vec{d},\; c = |\vec{o}-\vec{c}|^2 - r^2\)

Discriminant \(\Delta = b^2 - 4ac\); reject if \(\Delta < 0\)

Hit record:

  • \(t_\text{hit}\) - ray parameter at intersection
  • \(\vec{P}(t_\text{hit})\) - world-space point
  • \(\hat{n} = (\vec{P} - \vec{c})\,/\,r\) - outward normal

Self-intersection avoidance:

Offset shadow/reflection ray origin by \(10^{-4}\) along \(\hat{n}\) to avoid re-hitting the surface just left.

Spatial Acceleration: Amanatides & Woo  1987

Naïve intersection: \(O(n)\) per ray. Uniform voxel grid + DDA traversal: \(O(\sqrt[3]{n})\) average.

tMax[X] = ray parameter at which the ray next crosses an X-axis voxel boundary. tDelta[X] = how much \(t\) increases per voxel width along X. At each step we advance across the nearest boundary - guaranteed front-to-back order.

Initialization (slab test → entry voxel):

tMax[X] = (boundary - o.x) / d.x;
tDelta[X] = voxelSize / |d.x|;
// repeat for Y, Z

Traversal loop (2 cmp + 1 add per step):

if (tMax.x < tMax.y && tMax.x < tMax.z) {
    X += step.x;  tMax.x += tDelta.x;
} else if (tMax.y < tMax.z) {
    Y += step.y;  tMax.y += tDelta.y;
} else {
    Z += step.z;  tMax.z += tDelta.z;
}

Deduplication:

Sphere spanning multiple voxels → assign unique rayID; skip if already tested this ray.

Early stop:

If \(t_\text{hit} \leq \min(t_{\max X}, t_{\max Y}, t_{\max Z})\), hit is in current voxel - stop traversal.

Pros

  • Simple, cache-friendly
  • GPU-parallelisable
  • O(1) per step

Cons

  • Poor for non-uniform scenes
  • No empty-space skipping
  • BVH dominates production

Lambertian Diffuse  Lambert, 1760

Physical premise: perfectly matte surface — incident energy scattered equally in all directions over the hemisphere.

BRDF (constant):

\[ f_r = \frac{\rho}{\pi} \]

Rendered intensity:

\[ I = k_a + (1 - k_a)\max(0,\,\hat{N}\cdot\hat{L})\,v_\text{shadow} \]

\(k_a = 0.12\) (ambient), \(v_\text{shadow} \in \{0,1\}\)

Shadow test:

Vec3 shadowOrig = hit.point
                + hit.normal * 1e-4;
Ray  shadowRay(shadowOrig, L);
bool blocked = scene.trace(shadowRay)
               .has_value();

Normal-tinted color (our implementation):

// surface orientation proxy
r = intensity * (0.5 + 0.5 * n.x);
g = intensity * (0.3 + 0.5 * n.y);
b = intensity * (0.4 + 0.3 * n.z);

Lambert: Analysis

Baseline algorithm. Every more complex shading model reduces to Lambert when all fancy terms go to zero.

Pros

  • \(O(1)\) cost per pixel
  • Physically grounded BRDF
  • Zero tuning parameters
  • Energy-conserving (if \(\rho \leq 1\))
  • Good baseline for visual comparison

Cons

  • No specular highlights
  • Hard binary shadows - unphysical
  • No view-dependent effects
  • No inter-reflection
  • All surfaces look identically matte
State of the art Lambert: still the dominant diffuse model in production. PBR pipelines (e.g., Unreal, Unity) use it as the diffuse lobe in the split-sum approximation for real-time IBL.

Blinn-Phong: The Model

Half-vector:

\[ \hat{H} = \frac{\hat{L} + \hat{V}}{\|\hat{L} + \hat{V}\|} \]

\(\hat{V} = -\vec{d}_\text{ray}\) (toward camera)

Specular highlight:

\[ k_s = \max(0,\,\hat{N}\cdot\hat{H})^n, \quad n = 64 \]

Full model (\(\sigma = 0.6\)):

\[ I = \underbrace{k_a + (1-k_a)\max(0,\hat{N}{\cdot}\hat{L})}_{\text{Lambert term}} + \sigma k_s \]

Shininess \(n\) controls highlight sharpness: \(n=10\) → matte paint, \(n=64\) → glossy plastic (ours), \(n=1000+\) → near-mirror. Higher \(n\) concentrates the lobe around the mirror reflection direction.

// V points toward camera
Vec3 V = ray.direction * -1.0;
Vec3 H = (L + V).normalized();
double NdotH = max(0.0, N.dot(H));
double spec  = pow(NdotH, 64.0);

// final color per channel
r = diffuseIntensity*(0.5+0.5*n.x) + spec*0.6;
g = diffuseIntensity*(0.3+0.5*n.y) + spec*0.6;
b = diffuseIntensity*(0.4+0.3*n.z) + spec*0.6;

Why half-vector over reflection vector?
\(\vec{R} = 2(\hat{N}{\cdot}\hat{L})\hat{N} - \hat{L}\) requires a subtract + dot per pixel.
\(\hat{H}\) is a normalize; equivalent for directional lights and cheaper overall.

Blinn-Phong: Analysis

Pros

  • \(O(1)\) per pixel
  • View-dependent highlight
  • Perceptually convincing for plastics, metals
  • Hardware-accelerated in rasterization pipelines
  • Single tuning parameter (\(n\))

Cons

  • Not energy-conserving: \(\int f_r \cos\theta\,d\omega\) can exceed 1
  • No Fresnel: reflectivity constant at all angles
  • Shininess \(n\) is empirical, not physically meaningful
  • Hard shadows; no inter-reflection
  • Replaced by GGX in modern PBR pipelines
State of the art: Blinn-Phong was the GPU standard through the 2000s. Modern PBR (Physically Based Rendering) replaces it with the GGX microfacet BRDF (Walter et al., 2007) which correctly models the Fresnel term and energy conservation.

Soft Shadows: Distributed Ray Tracing  Cook, Porter, Carpenter, 1984

Problem: a point light casts a perfectly sharp shadow boundary - physically impossible for any real light source with nonzero area.

Physical model:
An area light of radius \(r\) at distance \(d\) subtends solid angle \(\approx \pi r^2 / d^2\). Points in the penumbra see a partial fraction of the light disk.

Cook's insight: distribute shadow rays stochastically over the light's solid angle. The fraction of unblocked rays estimates the partial visibility.

Penumbra width \(\propto r / d\). Same framework generalises to depth of field, motion blur, and glossy reflections.

Cook et al. (1984) unified stochastic sampling under a single framework: the shading integral \[I = \iint L(\phi_i,\theta_i)\,R(\phi_i,\theta_i,\phi_r,\theta_r)\,d\phi_i\,d\theta_i\] is estimated by distributing samples across all dimensions simultaneously.

Soft Shadows: Area Light Sampling Math

Uniform point on unit sphere (Marsaglia 1972):

Uniformly sample the unit disk (reject if outside), then project onto the sphere - avoids over-sampling the poles that naive spherical-coordinate sampling produces.

Reject \((x_1, x_2)\) if \(s = x_1^2 + x_2^2 \geq 1\), then:

\[ f = 2\sqrt{1-s}, \quad \vec{u} = (x_1 f,\; x_2 f,\; 1 - 2s) \]

Shadow factor (continuous penumbra):

\[ \text{sf} = \frac{\#\{\text{unblocked rays}\}}{N}, \quad N = 16 \]

Final intensity:

\[ I = k_a + (1-k_a)\,(\hat{N}{\cdot}\hat{L}_c)\cdot\text{sf} \]

Per-pixel deterministic seed:

Hash the primary ray direction into a seed. This means that there is no temporal flicker when navigating light positions.

for (int i = 0; i < N_SAMPLES; ++i) {
  Vec3 sample = light.pos
              + randomUnitVec(rng) * light.radius;
  Vec3 L = (sample - hit.point).normalized();
  if (N.dot(L) > 0 && !occluded(...))
      unblocked += 1.0;
}
double sf = unblocked / N_SAMPLES;

Soft Shadows: Analysis

Pros

  • Physically correct penumbra
  • Continuous shadow gradient
  • Simple to understand and implement
  • Penumbra naturally scales with light size

Cons

  • 16× slower than Lambert per pixel
  • Fixed \(N=16\) → residual noise at shadow edges
  • No importance sampling (uniform sphere)
  • Only diffuse; no specular term
  • No inter-reflection
State of the art: Production renderers use Multiple Importance Sampling (MIS) to combine BRDF sampling and light sampling optimally. ReSTIR (Bitterli et al., 2020) reuses reservoir samples across pixels and frames - achieving quality equivalent to sampling thousands of lights at the cost of ~1 sample per pixel.

Limits of Local Models: Path Notation

Shirley's regular-expression notation characterises which light transport paths each algorithm handles. \(E\) = eye, \(L\) = light, \(D\) = diffuse, \(G\) = glossy, \(S\) = specular.

Read a path left-to-right: camera (E), then a sequence of surface interactions, then light (L). \(S^*\) means zero or more specular bounces. \((D|G)\) means one diffuse or glossy hit.

AlgorithmPaths handledWhat is missed
Appel ray casting\(E(D|G)L\)everything beyond one bounce
Whitted RT\(E[S^*](D|G)L\)diffuse inter-reflection, caustics via diffuse
Radiosity (Goral)\(ED^*L\)specular effects entirely
Path Tracing\(E[(D|G|S)^+]L\)nothing (in principle)
Any algorithm that ignores whole classes of paths will produce incorrect images. They will produce wrong answers for scenes where those paths dominate (e.g., caustics, colour bleeding).

Whitted Ray Tracing  Whitted, 1980

Extends Phong shading with recursive specular reflection and refraction. First algorithm to correctly handle mirror inter-reflections.

Illumination model (we implement a simplified version of this):

\[ I = I_a + k_d\sum_j(\hat{N}{\cdot}\hat{L}_j) + k_s S + k_t T \]

\(S\) = recursive specular reflection, \(T\) = refraction

Reflection direction:

\[ \vec{R} = \vec{D} - 2(\vec{D}{\cdot}\hat{N})\hat{N} \]

Blending (our implementation, no refraction):

\[ C = C_\text{local}(1 - k) + C_\text{refl}\cdot k, \quad k = 0.4 \] 
// mirror reflection
Vec3 R = D - N * (2.0 * D.dot(N));
Ray  reflected(hit + N*1e-4, R);

Color refl = traceImpl(reflected,
                        scene, light,
                        depth - 1);
return local*(1.0-k) + refl*k;

Recursion terminates at depth == 0 or on scene miss.

Whitted: Implementation Details

Full trace function:

Color traceImpl(Ray ray, Scene& s,
                Light& l, int depth) {
  auto hit   = s.trace(ray);
  auto lhit  = l.intersect(ray);

  // light sphere visible?
  if (lhit && (!hit || lhit->t < hit->t))
    return {1,1,1};
  if (!hit)
    return background(ray);

  // local Blinn-Phong term
  Color local = BlinnPhong::shadeHit(
                  *hit, s, l, ray);
  if (depth <= 0) return local;

  // reflect
  Vec3 R = D - N*(2.0*D.dot(N));
  Ray  refl(hit->point + N*1e-4, R);
  Color rc = traceImpl(refl,s,l,depth-1);

  return local*0.6 + rc*0.4;
}

Parameters:

  • MAX_DEPTH = 4 (4 mirror bounces max)
  • REFLECTIVITY = 0.4 (uniform for all surfaces)
  • Local term: full Blinn-Phong (reuses BlinnPhong::shadeHit)

Our simplifications vs. Whitted's original:

  • No refraction (\(k_t = 0\))
  • Uniform reflectivity - no per-material \(k_s\)
  • Hard shadows (no distributed shadow rays)

Complexity:

Worst case \(O(2^d)\) rays per pixel (\(d\) = depth); in practice capped at 4 bounces = max 16 rays.

Whitted: Analysis

Pros

  • Captures mirror inter-reflections
  • Handles specular caustics (e.g., sphere reflecting light on wall)
  • Correct specular path chains \(S^*\)
  • Deterministic - no noise
  • Bounded cost (depth cap)

Cons

  • Misses all diffuse inter-reflection
  • Uniform reflectivity - all surfaces identical
  • Hard shadows (no penumbra)
  • Not energy-conserving
  • No refraction in our implementation
State of the art: Whitted-style tracing is still used for specular-heavy effects in hybrid renderers. In game engines, screen-space reflections (SSR) approximate it cheaply; RTX hardware enables full recursive tracing in real time.

What Whitted Misses: Diffuse Interreflection

Classic example: a white Cornell box with one red and one green wall. Adjacent white walls should pick up coloured light through multiple diffuse bounces.

Whitted traces \(E[S^*](D|G)L\): after one diffuse hit, tracing stops. The red/green bounce cannot reach the white walls - the image is simply wrong.

The solution: treat the integral in the rendering equation as a high-dimensional integral and solve it with Monte Carlo sampling.

Kajiya (1986): each bounce randomly samples a new direction from the BRDF. After many samples per pixel the estimator converges to the true rendering equation solution for all path types simultaneously.

Part 4

Monte Carlo Path Tracing

Rendering equation · Random walks · Russian Roulette · NEE

Rendering Equation: Area Form

Convert the hemisphere integral to an area integral over all surfaces via \(d\omega_i = \cos\theta'/|\vec{x}-\vec{x}'|^2\,dA'\):

\[ L_r(\vec{x}, \vec{\omega}) = \int_A f_r\!\left(\vec{x},\,\vec{\omega}(\vec{x},\vec{x}'),\,\vec{\omega}\right) L_o(\vec{x}', \vec{\omega}(\vec{x}',\vec{x}))\; G(\vec{x},\vec{x}')\; V(\vec{x},\vec{x}')\; dA' \]

\(G(\vec{x},\vec{x}') = \cos\theta_i\cos\theta'_o\,/\,|\vec{x}-\vec{x}'|^2\)  -  geometry factor (differential form factor / \(\pi\))

G bundles three physical factors: \(\cos\theta_i\) (foreshortening at the receiver), \(\cos\theta'_o\) (foreshortening at the source), and \(1/d^2\) (inverse-square falloff). \(V(\vec{x},\vec{x}')\) is 1 if the two points can see each other - implemented as a shadow ray test.

\(L\) appears on both sides → Fredholm integral equation of the second kind

\[ L = L_e + K \circ L \quad\Longrightarrow\quad L = (I-K)^{-1}\circ L_e = \sum_{n=0}^{\infty} K^n \circ L_e \]

Each term \(K^n \circ L_e\) = contribution of all light paths that make exactly \(n+2\) vertices (eye, \(n\) surface bounces, light).

Monte Carlo Integration

General Monte Carlo estimator:

\[ \int f(x)\,dx \approx \frac{1}{N}\sum_{i=1}^{N} \frac{f(x_i)}{p(x_i)}, \quad x_i \sim p \]

Variance \(\sigma^2 \propto 1/N\) → error \(O(1/\sqrt{N})\) regardless of dimension. This is crucial - the rendering integral is over paths of unbounded length.

Importance sampling:

Choose \(p \propto f\) to minimise variance. For Lambertian BRDF:

\[ p(\vec{\omega}_i) = \frac{\cos\theta_i}{\pi} \]

Cancels \(f_r\cos\theta_i\) in the estimator: weight per bounce = \(\rho\) only.

Cosine-weighted hemisphere sampling (Malley):

\[ r = \sqrt{u_1},\quad \phi = 2\pi u_2 \] \[ x = r\cos\phi,\quad y = r\sin\phi,\quad z = \sqrt{1-u_1} \]

Build ONB \((\vec{u},\vec{v},\hat{n})\) and return \(x\vec{u} + y\vec{v} + z\hat{n}\).

Verification:

\[ \frac{f_r \cos\theta}{p} = \frac{(\rho/\pi)\cos\theta}{\cos\theta/\pi} = \rho \;\checkmark \]

The Path Tracing Algorithm  Kajiya, 1986

Eye-ray random walk (forward tracing):

Step 1. weight \(= 1\), ray \(=\) camera ray
Step 2. Trace ray → nearest hit
Step 3A. \(L\) += weight \(\cdot L_e\) (emission)
Step 3B. weight \(*=\rho\); scatter by BRDF → Step 2

Russian Roulette termination (unbiased):

Continue with probability \(p = \max(\rho_r,\rho_g,\rho_b)\), clamp to \([0, 0.95]\).

\[ \text{weight} \leftarrow \text{weight} / p \quad\text{(compensate)} \]

Expected value unchanged; path terminated with correct probability.

Throughput = product of all BRDF weights so far (starts at 1, multiplied by \(\rho\) at each bounce). lastSpecular prevents double-counting: when NEE explicitly samples the light at a diffuse hit, we suppress the direct emission at the next hit to avoid adding it twice.

Color L = {0,0,0};
Color throughput = {1,1,1};

for (int d = 0; d < maxDepth; ++d) {
  auto hit = scene.trace(ray);
  if (!hit) { L += throughput*sky; break; }

  // accumulate emission
  if (lastSpecular)
    L += throughput * hit->mat->emission;

  // Russian Roulette
  double p = max(albedo.r,albedo.g,albedo.b);
  if (d >= rrStart && rng.next() > p) break;
  if (d >= rrStart) throughput *= 1.0/p;

  throughput = hadamard(throughput, albedo);
  ray = scatter(hit, ray, rng);
}

BRDF Sampling

Lambertian (\(f_r = \rho/\pi\)):

Cosine-weighted hemisphere → weight = \(\rho\) (shown on previous slide).

// Malley's method
double r   = sqrt(u1);
double phi = 2.0*M_PI*u2;
double x = r*cos(phi), y = r*sin(phi);
double z = sqrt(max(0.0, 1.0 - u1));
// transform to world via ONB(normal)

Mirror (\(f_r = \rho\,\delta(\vec{\omega}_i - \vec{R})\)):

\[ \vec{R} = \vec{D} - 2(\vec{D}{\cdot}\hat{N})\hat{N} \]

Delta function: pdf = \(\infty\) at \(\vec{R}\), zero elsewhere. Estimator weight still = \(\rho\).

Orthonormal basis construction:

void ONB(Vec3 n, Vec3& u, Vec3& v) {
  Vec3 a = (abs(n.x) > 0.9)
         ? Vec3{0,1,0} : Vec3{1,0,0};
  v = n.cross(a).normalized();
  u = n.cross(v);
  // u,v,n are orthonormal, right-handed
}

Next Event Estimation (NEE)

Core idea: instead of hoping the random walk accidentally finds the light, shoot a dedicated shadow ray directly toward it at every diffuse bounce. The contribution is computed analytically rather than discovered by chance - dramatically reducing variance.

Why it works: the rendering equation contains a direct-lighting integral we can evaluate exactly. NEE evaluates it explicitly; the path continues for indirect light only.

Direct lighting estimator (area form):

\[ L_\text{direct} = \frac{f_r \cdot L_e \cdot \cos\theta_\text{hit} \cdot \cos\theta_\text{light} \cdot A_\text{light}}{d^2} \]

\(A_\text{light} = 4\pi r^2\), \(\;p_\text{area} = 1/A\), \(\;f_r = \rho/\pi\)

After folding \(\rho\) into throughput:

\[ \text{weight} = \frac{\cos\theta_\text{hit}\cdot\cos\theta_\text{light}\cdot A}{\pi\, d^2} \]

Double-counting avoidance:

After a diffuse bounce, \(L_e\) is suppressed at the next hit (NEE already counted it). Only camera rays and post-specular hits add \(L_e\) directly. → lastSpecular flag.

// at each diffuse hit:
for (Sphere* light : scene.lights()) {
  Vec3 lPt = light->center
           + uniformSphere(rng)*light->radius;
  Vec3 dir = (lPt - hit.point).normalized();
  double dist = (lPt - hit.point).norm();

  if (N.dot(dir) <= 0) continue;
  Vec3 lN = (lPt-light->center)/light->radius;
  if (lN.dot(-dir) <= 0) continue;  // back face
  if (occluded(scene, hit, dir, dist)) continue;

  double A = 4*M_PI*light->radius*light->radius;
  double w = N.dot(dir) * lN.dot(-dir)
           * A / (M_PI * dist*dist);
  L += hadamard(throughput, light->emission)*w;
}

Tone Mapping & Display

Path-traced radiance is HDR (unbounded). Display is 8-bit sRGB (\([0, 255]\)). Need a mapping that:

  • Compresses highlights without clipping
  • Preserves relative luminance in midtones
  • Approximates perceptual linearity

Reinhard operator:

\[ x' = \frac{x}{1 + x} \]

Maps \([0, \infty) \to [0, 1)\). Asymptotically correct behaviour for overbright HDR.

Gamma 2.2:

\[ x'' = (x')^{1/2.2} \]

Linearises perceptual response of display. Combined:

int toSRGB(double x) {
  x = x / (1.0 + x);          // Reinhard
  x = pow(x, 1.0/2.2);        // gamma
  return clamp((int)(x*255.999), 0, 255);
}

Progressive refinement:

Accumulate per-pixel sums \(\sum L\); display running average \(\sum L / \text{spp}\) after each sample pass. Image visibly refines in real time.

sumR[idx] += L.r;
// ...
double inv = 1.0 / (double)spp;
display.setPixel(col, row,
  toSRGB(sumR[idx]*inv),
  toSRGB(sumG[idx]*inv),
  toSRGB(sumB[idx]*inv));

Variance \(\propto 1/\text{spp}\) → doubling spp halves noise standard deviation. Clean image typically requires 256–4096 spp for our scenes.

Path Tracing: Analysis

Pros

  • Unbiased estimator of the full rendering equation
  • Handles all light transport: colour bleeding, caustics, interreflection
  • Physically correct - no ad hoc parameters
  • Progressive: improves continuously with more samples
  • Naturally handles area lights, environment maps

Cons

  • Slow: error \(O(1/\sqrt{N})\); need \(4\times\) samples to halve noise
  • Fireflies: rare high-energy paths produce bright outlier pixels
  • Hard to debug - noise obscures visual artefacts
  • Caustics (\(ES^+DL\)) converge especially slowly
  • Glossy BRDF not implemented (only diffuse + mirror)
NEE dramatically reduces variance for scenes with visible emitters at the cost of one shadow ray per diffuse hit per light. The improvement is most visible at low spp (early convergence).

State of the Art

RTX · ReSTIR · AI Denoising · Production renderers

Hardware Ray Tracing: NVIDIA RTX  2018–present

  • RT Cores: fixed-function hardware for BVH traversal and ray–triangle intersection. Offloads the innermost loop of our voxel traversal entirely.
  • Two-level BVH: BLAS per mesh (geometry), TLAS over all instances. Replaces our uniform grid; O(log n) traversal for any scene layout.
  • Performance: ≈10 Giga-rays/sec (RTX 4090) vs ≈100M/sec software → enables real-time path tracing at 1080p/30fps.
  • API: DXR (DirectX 12) / Vulkan RT - programmable shaders for ray-gen, closest-hit, any-hit, miss.

BVH vs our uniform grid:
Our voxel grid: O(1) per step, but must step through many empty voxels in non-uniform scenes.

SAH-split BVH: O(log n) traversal; adapts to scene density; standard in all production renderers since ~2000.

What stays the same: the shading and BRDF evaluation we implemented runs identically on the GPU - RT Cores only accelerate the intersection step.

ReSTIR  Bitterli et al., 2020

Reservoir-based Spatiotemporal Importance Resampling - solves the many-light problem in NEE.

Problem: our NEE casts one shadow ray per light per pixel. With thousands of lights, this is prohibitive.

Weighted Reservoir Sampling: maintain a size-\(M\) reservoir of light candidates. New candidate \(x\) accepted with probability \(w(x)/W_\text{total}\). After \(N \gg M\) candidates, reservoir contains \(M\) samples proportional to target distribution.

Spatial reuse: neighbouring pixels share reservoirs → effectively samples many more lights.

Temporal reuse: previous frame's reservoir reused → amortises cost over time.

Quality equivalent to sampling hundreds of lights per pixel, at cost of 1–2 shadow rays per pixel per frame.

AI Denoising & DLSS

Problem: path tracing at 1–4 spp is visibly noisy (as you'll see in our demo). Traditional denoising (bilateral filter, NL-means) smooths but blurs detail.

NVIDIA OptiX AI Denoiser (2017): CNN trained on HDR Monte Carlo renders.

DLSS 3.5 Ray Reconstruction (2023): joint denoising + temporal upscaling in one network. Rendered at 1080p with 1 spp → displayed at 4K with perceived quality of many spp.

Why CNNs work here:
Monte Carlo noise is high-frequency, spatially correlated, and lies on top of low-frequency signal. This is exactly the signal/noise structure that convolutional networks handle well, especially with temporal coherence as additional context.
